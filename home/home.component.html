Deploying Angular Client to Microsoft Azure from Local GIT Repo

- Daniel Cassin

In this article I demonstrate how to deploy an angular client app from a local git repo to an Azure App Service using the Azure Deployment Center

{{isDetails1 ? 'READ MORE' : 'READ LESS'}}

1 - In Azure Portal

  1. Create a new Azure App Service (Create Web App)
  2. Once it is finished deploying click (upper right of screen) Go to Resource button
  3. Click on Deployment Center (on left menu under Deployment)
  4. Select Local Git
  5. Select App Service build service box and click Continue
  6. Click Finish
  7. Copy the uri from the Git Clone Uri (ex: https://danielcassin.scm.azurewebsites.net:443/danielcassin.git)
  8. Click on Deployment Credentials and setup/copy username & password as you will need it for git local repository

2 - In Angular CLI

  1. In order to create / update a dist folder which you will use for deployment to production enter the following command:
    ng build --prod
  2. In File Explorer create a new folder above the project name (../Azure)
  3. Inside this folder (in npm), create a new git repo with the following command:
    git init
  4. Copy the contents of the dist folder into this new folder
  5. Add files to the git repo:
    git add *.*
  6. Commit the new changes:
    git commit --message "initial checkin"
  7. Push changes to Azure app service:
    git remote add origin https://dcassin234@jungle-client.scm.azurewebsites.net:443/jungle-client.git
  8. After Initial Check in (anytime you want to make changes) just repeat as follows
    1. ng build --prod (copy/paste dist folder contents into azure folder)
    2. git add *.*
    3. git commit -m(essage) “some message”
    4. git push origin master
Azure Dev Ops

- Daniel Cassin

In this article I show how to set up an Azure SQL Database, publish a .NET Core API from Visual Studio to an Azure App Service, and create an Azure Pipleline with Azure DevOps Starter

{{isDetails2 ? 'READ MORE' : 'READ LESS'}}

Setting up the Azure SQL Database
  1. From Azure Portal Favorites menu, select SQL databases
  2. Click Add button
  3. Create New Resource Group, give it a name
  4. Select database name from suggestions
  5. Under select a server, click Create New
  6. Need to set up IP before we can deploy local db to azure
    • Go to Firewall Settings and click an Add Client IP
    • Click Save button near top
  7. Go to SQL Server Management Studio
    • Find the database to deploy and right click
    • Select tasks > Deploy to microsoft azure …
    • Enter the azure sql server name and login credentials
Publishing the Web API From Visual Studio
  1. From Home, Click on App Services
  2. Click Create app service button
  3. Select resource group (or Create New)
  4. Give it a name (jungleapi)
  5. Select runtime stack (.NET Core 3.1
  6. Select region (West US)
  7. Click review & create button
  8. Click Create button … Deployment in progress
  9. After deployment done (Your deployment is complete), then click Go to resource
    here you will see your URL, App service plan, etc..
  10. In visual studio right click api project and click publish
  11. Select resource group, service name
  12. Click Edit link then Validate Connection
  13. Click next, then save
  14. Click publish button
Create Azure pipeline for .NET with Azure DevOps Starter
  1. Sign in to Azure portal
  2. From home page type DevOps Starter in search box
  3. Click on Create DevOps starter button
  4. Select .NET from the applications buttons menu
  5. Select ASP.NET Core
  6. Select Windows Web App
  7. Enter project name (ex: JunglePipeline) & Azure DevOps Organization (ex: JungleOrganization)
  8. Select Location (West)
  9. Click Done button
  10. After deployment is complete click Go to resource
